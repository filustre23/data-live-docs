# 瞭解 BigQuery 版本

BigQuery 提供三種版本，支援不同類型的工作負載和相關功能。您可以在[預留 BigQuery 容量](https://docs.cloud.google.com/bigquery/docs/reservations-intro?hl=zh-tw#reservations)時啟用版本。BigQuery 也提供[隨選 (每處理 1 TiB) 模式](https://cloud.google.com/bigquery/pricing?hl=zh-tw#on_demand_pricing)。您可以選擇在每個專案中同時使用版本和隨選模式。如要進一步瞭解 BigQuery 版本定價，請參閱 [BigQuery 定價](https://cloud.google.com/bigquery/pricing?hl=zh-tw)。

每個版本都提供不同價位的一系列功能，可滿足不同類型機構的需求。您可以建立與版本相關聯的[預留項目](https://docs.cloud.google.com/bigquery/docs/reservations-intro?hl=zh-tw)或[容量承諾](https://docs.cloud.google.com/bigquery/docs/reservations-details?hl=zh-tw#commitments)。如要變更與預訂項目相關聯的版本，請刪除預訂項目，然後使用新版本類型重新建立。詳情請參閱「[更新預訂](https://docs.cloud.google.com/bigquery/docs/reservations-tasks?hl=zh-tw#update_reservations)」。設定為使用[運算單元自動調度資源](https://docs.cloud.google.com/bigquery/docs/slots-autoscaling-intro?hl=zh-tw)的預留項目會自動調度資源，以滿足工作負載的需求。購買運算單元時，不一定要簽訂容量承諾，但這樣做可以降低費用。由於 BigQuery 版本是運算能力的屬性，而非儲存空間，因此無論資料集儲存方式為何，只要版本支援您想使用的功能，您都可以查詢資料集。所有版本的配額都相同。您的配額並非以版本為單位計算。如要進一步瞭解配額，請參閱「[配額與限制](https://docs.cloud.google.com/bigquery/quotas?hl=zh-tw#reservations)」。

## BigQuery 版本功能

下表列出各版本支援的功能。系統會封鎖版本以外的功能，或缺少相關功能。

請勿使用版本層級限制特定功能的存取權，因為指派給各個版本的功能可能會隨時間變更。舉例來說，請勿將專案指派給 Standard 版預留項目，藉此禁止存取 BigQuery ML。
