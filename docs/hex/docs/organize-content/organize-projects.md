# Organize projects

Organize your workspace by updating projects individually or in bulk.

## Filter and sort projects[​](#filter-and-sort-projects "Direct link to Filter and sort projects")

When viewing the **Projects** page, all users in a workspace are able to filter the list of projects to find relevant projects. Users can filter by project title, status, categories, and a number of other fields including the project owner and your level of access.

[](/assets/medias/filter-projects-4dca147d98ba3448506172711db3081c.mp4)

Additionally, users can sort this list on a number of attributes, including when the project was last edited, the number of app views, or when the app was last published.

## Organize projects[​](#organize-projects "Direct link to Organize projects")

[Admins](/docs/collaborate/sharing-and-permissions/roles), [Managers](/docs/collaborate/sharing-and-permissions/roles), and [Editors](/docs/collaborate/sharing-and-permissions/roles) are able to organize projects from the **Projects** page, or any other home page where a list of projects is rendered.

[](/assets/medias/trash-projects-656395b545c69db61272414eee313b06.mp4)

To organize projects:

1. Select a project:
   * If viewing the list of projects in table view, select the checkbox next to the project name
   * If viewing the list of projects in card view, hold down `command` (or `ctrl`) and click the card
2. Select additional projects. You can do this by individually selecting more projects (as above), or using `shift + click` to select a range of projects
3. Choose the action you'd like to perform to organize the projects

tip

If you are an [Editor](/docs/collaborate/sharing-and-permissions/roles) and select a project that you don't have the right level of access to update, some actions will be disabled. Use the **Update selection** button to refine your selection.

[](/assets/medias/update-selection-79d183edc4b190857c183b1dc2cde4c4.mp4)

Projects can also be updated individually, either from the **Projects** page, via the three-dot menu next to to the project name, or **when editing a project** via the menu next to the project name.

## Available actions[​](#available-actions "Direct link to Available actions")

The following actions are available from the **Projects** page:

| Action | Access required |
| --- | --- |
| [Change status](/docs/organize-content/statuses-categories) | Admin or Manager role, or **Can edit** access |
| [Change categories](/docs/organize-content/statuses-categories) | Admin or Manager role, or **Can edit** access |
| [Add to, or remove from Collections](/docs/organize-content/collections) | Admin or Manager role, or **Can explore** access |
| [Move to, or restore from Archive](/docs/organize-content/archive) | Admin or Manager role, or **Full access** |
| Trash | Admin or Manager role, or **Full access** |
| Add to, or remove from Favorites | Any access |
| [Transfer owner](/docs/organize-content/project-owners) | Admin or Manager role, or **Full access** |
| [Require reviews](/docs/collaborate/reviews) | Admin or Manager role, or **Full access** |
