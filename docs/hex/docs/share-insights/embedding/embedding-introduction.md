# Embedding introduction

Hex supports public, private, and signed embedding for a wide range of embedded analytics use cases.

## What is embedding?[​](#what-is-embedding "Direct link to What is embedding?")

Embedding refers to integrating data apps, visualizations, dashboards, or reports into a website or web application.

## Different types of embedding[​](#different-types-of-embedding "Direct link to Different types of embedding")

### Public embedding[​](#public-embedding "Direct link to Public embedding")

Public embedding allows you to embed Hex apps that are shared publicly and require no authentication. This type of embedding is ideal for content meant to be accessible to anyone, such as embedding public data on a marketing site. [Learn more](/docs/share-insights/embedding/public-and-private-embedding#public-embedding).

### Private embedding[​](#private-embedding "Direct link to Private embedding")

Private embedding allows you to embed Hex apps that are shared privately and require authentication into your Hex workspace to view. This type of embedding is a good fit for internal use cases, such as embedding company data in an internal tool for employees. [Learn more](/docs/share-insights/embedding/public-and-private-embedding#private-embedding).

### Signed embedding (embedded analytics)[​](#signed-embedding-embedded-analytics "Direct link to Signed embedding (embedded analytics)")

Our embedded analytics product offering supports signed embedding, which allows customers to embed Hex apps securely and seamlessly in their web applications with pass-through authentication and row-level security. This type of embedding is ideal for external audience use cases with secure content that may need to be filtered or restricted per user. Common use cases include building customer-facing analytics into your SaaS product, or adding monetizable data reports to your client portal. [Learn more](/docs/share-insights/embedding/signed-embedding).
